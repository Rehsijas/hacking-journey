Session1 - The terminal 
-> the terminal is just a way to talk to your computer through text insted of clicking (nothing magical)

->when you see this on your terminal
(for example: jasmeetrehsi@Mac hacking-journey %)
its telling you three things- who you are, which machine, and where you are
the % symbol means the terminal is waiting for your command 


->below given are few commands written one by one and next to them are explaination in my own words about what it does or perform 
1) whoam1 :- it gonna return your name , for example-"jasmeetrehsi" in my case 
2) pwd:- it means print working directory, for example- it returns "/users/jasmeetrehsi/hacking-journey" 
3)ls:- it means lists, for example- README.md
4)ls -la:- it returns ''' total8
drwxr-xr-x@ 4 jasmeetrehsi staff 128 feb25 11.13.
drwxr-x---+ 36 jasmeetrehsi staff 452 feb25 11.13..
drwxr-xr-x@ 13 jasmeetrehsi staff 416 feb25 11.16.git
-rw-r--r--@ 1 jasmmeetrehsi staff 573 feb25 11.15 README.md''', where drwxr.xr-x are permissions where d=directory, -=file, r=read, w=write, x=execute
5)cd:- it change directory
6)cd..:- here ".." is "one level up" and it takes you back to home
7)mkdir:- it is used for making directories
8)cat README.md:- cat means concatenate and it prints whatever is inside README>md  
9)touch testfile.txt:- it creates an empty file if its non existing, otherwise it just update timestamp
10)echo "hello Rehsijas" > testfile.txt :- writes "hello Rehsijas" in the file
11)rm testfile.txt:- deletes the file 
12)cp testfilet.txt backup.text:- copies the file 
13)mv testfile.txt renamed.txt:- move or renames the file
